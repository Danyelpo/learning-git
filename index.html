Learning Git

What is Git?

Git is a version control system that tracks changes in files. It allows returning to previous versions, working safely, and collaborating with others.

Why Git is Important?

  • Tracks code history
  • Prevents losing work
  • Enables teamwork
  • Industry standard for developers

Git basic commands

1. git init - Initialized empty Git repository in "user file path/.git/".

What happens when git init is executed?

  • A hidden .git folder is created
  • This folder contains:
    • Commit history
    • Branch information
    • Configuration files
  • The project becomes a Git repository

Note: Git does not track files automatically. It only starts tracking files after they are added and committed.

2. git status - It will show the staging area and working directory's current conditions.

What git status shows?

  • Current branch (usually main)
  • Files that are:
    • Untracked → new files Git doesn’t know about
    • Modified → changed files
    • Staged → ready to be committed

3. git add . or git add <file> - Using a dot (.) stages all modified and untracked files in the current directory and its subdirectories, while using a file name stages only that file.

What git add . does?

  • Adds new untracked files
  • Stages modified files
